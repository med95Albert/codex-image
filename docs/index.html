WHY・為什麼不直接在 ChatGPT 生圖

差在「脈絡」與「正確性」

  • 風格一致——你在專案根目錄放一份 STYLE.md（品牌色、字體、插畫風格、禁用元素），每次生圖自動注入。ChatGPT 網頁每次都要重新描述，還是會飄。
  • 專案脈絡——Claude Code 讀得到你的資料夾、文件、AGENTS.md，下給 Codex 的 prompt 比你手打的更精準。
  • 像素級正確——QR code、圖表、精確中文排版這類東西，指令強制 Codex「用程式生成」（qrcode／matplotlib），不是 AI 繪圖近似。差一個像素 QR code 就連不上，這條路保證能掃、不亂碼。

AI 繪圖畫的 QR code 是「長得像 QR code 的圖」；程式生成的 QR code 是真的連得上你網址的 QR code。

COST・省錢原理

同一份訂閱，兩個池子

  • 吃的是 agentic 用量——Codex CLI 生圖消耗訂閱方案的 agentic 額度，跟 ChatGPT 網頁版「每日幾張圖」是不同的池子，也完全不經過按量付費 API。
  • 生圖比文字燒得快——約 3～5 倍。偶爾生一兩張幾乎無感；連續大量就會見底。
  • 決策規則——少量、要精準／一致 → 走這條（訂閱）。大量批次（數十張以上）→ 改用 OpenAI 按量付費 API。

INSTALL・怎麼安裝

一句話就裝好

打開 Claude Code，把下面這句話貼給它——clone、放對位置、環境檢查，全部由 Claude 代勞：

幫我把 github.com/med95Albert/codex-image 這個技能裝到 ~/.claude/skills/codex-image，然後執行裡面的 install.sh 完成安裝檢查

裝完重開一個 Claude Code session 即生效。偏好原生外掛的話：/plugin marketplace add med95Albert/codex-image → /plugin install codex-image@codex-image。前置需求：Claude Code＋Codex CLI（brew install codex）＋ChatGPT Plus 以上訂閱（codex login 用 ChatGPT 帳號，別貼 API key）。原始碼與完整說明在 GitHub。

誠實的三個限制

  • 要養 ChatGPT 訂閱——Plus 以上且 Codex CLI 已登入；生圖跟你其他 Codex 任務搶同一份 agentic 額度。
  • AI 繪圖仍有隨機性——STYLE.md 大幅收斂風格，但 gpt-image 不保證每次一模一樣；只有程式化元素（QR code／圖表）有像素級保證。
  • 不適合大批次——數十張以上會撞訂閱上限、要一直等；那種量請改按量付費 API，skill 也會這樣勸你。
